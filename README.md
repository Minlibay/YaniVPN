# YaniVPN

Собственный VPN-сервис: веб-панель управления + агент для нод.
Два протокола на выбор для каждого сервера:

- **WireGuard** — быстрый, лучший для мобильных клиентов;
- **VLESS + Reality (Xray)** — обход блокировок: трафик маскируется под
  обычный HTTPS к реальному сайту, поэтому DPI не отличает его от посещения
  этого сайта.

Плюс мобильное приложение на Flutter (iOS/Android) — в папке
[`mobile/`](mobile/README.md).

## Что умеет панель

- **Дашборд** — серверы онлайн, активные подключения, суммарный трафик,
  графики трафика и подключений за 24 часа.
- **Серверы** — автоустановка: выбираете протокол, указываете IP и SSH-доступ
  root, панель сама ставит WireGuard или Xray, настраивает сеть, запускает
  агента и получает ключи ноды. Статусы установка/ошибка/онлайн/офлайн,
  повторная установка с логом ошибки. SSH-данные используются один раз и не
  сохраняются.
- **Клиенты** — выпуск клиентов под протокол сервера: WireGuard — пара ключей +
  адрес в туннеле и `.conf`; VLESS — UUID и ссылка `vless://…`. Для обоих —
  QR-код для импорта на телефон. Секреты клиента на сервере не хранятся.
  Клиентов можно отключать/включать и удалять — агент применяет изменения в
  течение минуты.
- **API для агентов** — каждая нода авторизуется собственным токеном и
  раз в минуту присылает статистику (`/api/agent/report`).

## Мобильное приложение

Flutter-клиент в [`mobile/`](mobile/README.md): без регистрации (личный
конфиг-код, переносимый между устройствами), список серверов и подключение,
бесплатный лимит 2 ГБ с экраном покупки доступа. Опирается на публичный
**API приложения** (`/api/app/*`, авторизация по коду аккаунта через
`Authorization: Bearer <код>`):

| Метод | Назначение |
|-------|-----------|
| `POST /api/app/register` | создать анонимный аккаунт, получить конфиг-код |
| `POST /api/app/session` | вход по коду с другого устройства |
| `GET /api/app/servers` | список доступных серверов |
| `POST /api/app/connect` | выдать конфиг подключения (учитывает квоту, `402` при исчерпании) |
| `GET /api/app/usage` | трафик и план аккаунта |
| `POST /api/app/upgrade` | оплата (заглушка; включается `ALLOW_MOCK_PURCHASE=1`) |

Пиры, выпущенные приложением, привязаны к аккаунту; квота — суммарный трафик
пиров аккаунта. При исчерпании лимита пиры отключаются, и агент убирает их с
ноды при следующей синхронизации.

## Стек

- [Next.js 14](https://nextjs.org) (App Router) + TypeScript + Tailwind CSS
- [Prisma](https://prisma.io) + PostgreSQL
- Recharts для графиков, jose для сессий, bcryptjs для паролей
- Продакшен: Docker Compose (панель + PostgreSQL + Caddy c автоматическим HTTPS)

## Быстрый старт

```bash
cp .env.example .env        # задайте AUTH_SECRET и пароль админа
# PostgreSQL для разработки (или свой локальный):
docker run -d --name yanivpn-db -p 5432:5432 \
  -e POSTGRES_DB=yanivpn -e POSTGRES_USER=yanivpn -e POSTGRES_PASSWORD=yanivpn \
  postgres:16-alpine
npm install
npx prisma db push          # создать базу
SEED_DEMO=1 npm run db:seed # админ + демо-данные (без флага — только админ)
npm run dev                 # http://localhost:3000
```

На Windows PowerShell: `$env:SEED_DEMO=1; npm run db:seed`.

Вход по умолчанию: `admin@yanivpn.local` / `admin123`
(переопределяется через `ADMIN_EMAIL` / `ADMIN_PASSWORD` до запуска сида).

Продакшен-деплой — Docker Compose с PostgreSQL и автоматическим HTTPS
(Let's Encrypt): [deploy/docker.md](deploy/docker.md). Старый вариант
(systemd, HTTP по IP) — [deploy/README.md](deploy/README.md).

## Подключение реальной ноды

Возьмите чистый VPS на Debian/Ubuntu и в панели нажмите **Серверы →
Добавить сервер**: выберите протокол, укажите IP и SSH-доступ root (пароль или
ключ). Панель подключится по SSH и за 1–3 минуты сама всё настроит:

- **WireGuard**: ставит `wireguard`, поднимает `wg0` (10.8.0.1/24, NAT,
  форвардинг), сохраняет публичный ключ ноды.
- **VLESS**: ставит Xray-core, генерирует ключи Reality, пишет `config.json`
  (маскировка под указанный SNI, порт 443), включает статистику по клиентам.

В обоих случаях агент устанавливается systemd-сервисом с токеном ноды. После
этого сервер станет «онлайн», а клиенты, созданные в панели, применяются к ноде
автоматически (WireGuard — через `wg set`, VLESS — переписыванием `clients` в
конфиге Xray). В продакшене задайте `PANEL_URL` в `.env` — этот адрес
прописывается агентам. Ручная установка агента по-прежнему возможна:
[agent/README.md](agent/README.md).

Полный прогон «настоящая нода + приложение на телефоне, реальный туннель» —
в [docs/real-connection.md](docs/real-connection.md).

## Структура

```
prisma/          схема БД и сид с демо-данными
agent/           агент для VPN-нод (bash + systemd)
src/app/         страницы панели и API-маршруты
  api/agent/     приём статистики от нод
  api/servers/   CRUD серверов
  api/peers/     выпуск и управление клиентами
src/components/  графики, формы, таблицы
src/lib/         auth (JWT-сессии), wg + vless (ключи/ссылки), provision (SSH), prisma
```
