# YaniVPN

Собственный VPN-сервис на WireGuard: веб-панель управления + агент для нод.
Мобильное приложение — следующий этап (архитектура API уже рассчитана на него).

## Что умеет панель

- **Дашборд** — серверы онлайн, активные подключения, суммарный трафик,
  графики трафика и подключений за 24 часа.
- **Серверы** — добавление/удаление VPN-нод, статус (онлайн/офлайн по отчётам
  агента), активные пиры, трафик, токен агента.
- **Клиенты** — выпуск клиентов WireGuard: панель генерирует пару ключей,
  выделяет адрес в туннельной подсети и отдаёт готовый `.conf`
  (приватный ключ клиента на сервере не хранится). Клиентов можно
  отключать/включать и удалять — агент на ноде применяет изменения в течение минуты.
- **API для агентов** — каждая нода авторизуется собственным токеном и
  раз в минуту присылает статистику (`/api/agent/report`).

## Стек

- [Next.js 14](https://nextjs.org) (App Router) + TypeScript + Tailwind CSS
- [Prisma](https://prisma.io) + SQLite (для продакшена меняется на PostgreSQL
  правкой `datasource` в `prisma/schema.prisma` и `DATABASE_URL`)
- Recharts для графиков, jose для сессий, bcryptjs для паролей

## Быстрый старт

```bash
cp .env.example .env        # задайте AUTH_SECRET и пароль админа
npm install
npx prisma db push          # создать базу
npm run db:seed             # админ + демо-данные
npm run dev                 # http://localhost:3000
```

Вход по умолчанию: `admin@yanivpn.local` / `admin123`
(переопределяется через `ADMIN_EMAIL` / `ADMIN_PASSWORD` до запуска сида).

## Подключение реальной ноды

Краткая версия (подробно — в [agent/README.md](agent/README.md)):

1. Поднимите WireGuard на сервере (`wg0`).
2. В панели добавьте сервер, указав его публичный ключ — получите токен агента.
3. Запустите на ноде `agent/yanivpn-agent.sh` (systemd-сервис) с этим токеном.
4. Через минуту сервер станет «онлайн»; клиентов создавайте в панели —
   агент сам применит их к WireGuard.

## Структура

```
prisma/          схема БД и сид с демо-данными
agent/           агент для VPN-нод (bash + systemd)
src/app/         страницы панели и API-маршруты
  api/agent/     приём статистики от нод
  api/servers/   CRUD серверов
  api/peers/     выпуск и управление клиентами
src/components/  графики, формы, таблицы
src/lib/         auth (JWT-сессии), wg (ключи, конфиги), prisma
```
