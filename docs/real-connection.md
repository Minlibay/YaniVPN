# Реальное подключение: от нуля до туннеля на телефоне

Пошаговый прогон «оно правда подключается»: настоящая нода (VPS) + приложение
на Android, туннель WireGuard. Панель при этом крутится у вас локально, но её
нужно сделать доступной из интернета — иначе агент на ноде и телефон до неё не
достучатся.

## Что понадобится

- **VPS** на Debian/Ubuntu с доступом по SSH под root (любой провайдер).
- **Панель** запущена локально (`npm run dev`).
- **cloudflared** — бесплатный туннель, чтобы отдать локальную панель в интернет
  без домена и настройки HTTPS. Скачивается одним файлом с сайта Cloudflare.
- **Android-эмулятор** (из Android Studio) или реальный телефон с Android.

## Почему нельзя просто localhost

Подключение зависит от двух связей с панелью:

1. **Агент на ноде → панель.** Когда приложение жмёт «Подключить», панель
   создаёт пир в своей базе, но в реальный WireGuard на ноде его добавляет
   **агент**, опрашивая панель. Если панель на `localhost`, агент с VPS её не
   видит → пир не появится на ноде → рукопожатие не пройдёт.
2. **Телефон → панель** за списком серверов и конфигом.

Поэтому панель выставляем наружу через cloudflared и этот публичный адрес
используем везде.

## Шаги

### 1. Выставить панель в интернет

В отдельном окне (панель уже запущена на `:3000`):

```bash
cloudflared tunnel --url http://localhost:3000
```

cloudflared напечатает адрес вида `https://random-words.trycloudflare.com`.
Дальше он обозначен как `PANEL_PUBLIC_URL`.

### 2. Перезапустить панель с этим адресом

Агентам нод нужно знать, куда слать отчёты. Остановите `npm run dev` и запустите
с `PANEL_URL` (Windows PowerShell):

```powershell
$env:PANEL_URL="https://random-words.trycloudflare.com"
npm run dev
```

(Оставьте cloudflared из шага 1 работать.)

### 3. Добавить настоящую ноду

В панели (через публичный адрес или `localhost:3000`) → **Серверы → Добавить
сервер**: протокол **WireGuard**, IP вашего VPS, SSH-доступ root. Панель по SSH
поставит WireGuard, откроет UDP-порт в `ufw`, запустит агента с
`PANEL_URL=PANEL_PUBLIC_URL`. Через 1–3 минуты сервер станет **онлайн** — значит,
агент достучался до панели (связь #1 работает).

> На время теста можно ускорить синхронизацию: агент берёт интервал из
> переменной `INTERVAL` (сек). По умолчанию 60. Чтобы новый клиент применялся
> быстрее, на ноде допишите `INTERVAL=10` в `/etc/yanivpn-agent.env` и
> `systemctl restart yanivpn-agent`.

Если у провайдера есть внешний firewall (Hetzner Cloud Firewall, AWS Security
Group и т.п.) — откройте там входящий **UDP 51820**: `ufw` внутри VPS этого не
покрывает.

### 4. Запустить приложение против публичной панели

```powershell
cd mobile
flutter run -d emulator --dart-define=API_BASE=https://random-words.trycloudflare.com
```

(Список эмуляторов — `flutter emulators`, запуск — `flutter emulators --launch <id>`.)

### 5. Подключиться

В приложении → сервер → **Подключить**. Android спросит разрешение на VPN —
подтвердите. Через несколько секунд (до `INTERVAL`, пока агент добавит пир)
поднимется туннель.

## Проверка, что реально работает

- В приложении статус сервера сменится на подключённый.
- На ноде: `wg show wg0` покажет ваш пир с недавним `latest handshake` и
  растущими `transfer`.
- В панели → **Клиенты** у клиента появится трафик и «в сети».
- Проверьте выходной IP: на телефоне в браузере откройте сайт «мой IP» — должен
  показать IP вашего VPS.

## Если не подключается

| Симптом | Причина | Что проверить |
|---|---|---|
| Сервер не становится «онлайн» | агент не достучался до панели | cloudflared жив? `PANEL_URL` публичный? `journalctl -u yanivpn-agent -f` на ноде |
| Туннель поднялся, но нет интернета / нет handshake | закрыт UDP-порт | внешний firewall провайдера на UDP 51820; `ufw status` на ноде |
| «Подключить» не даёт эффекта пару минут | пир ещё не синхронизирован | подождите `INTERVAL` или уменьшите его; `wg show wg0` на ноде — есть ли пир |
| Приложение не грузит серверы | не тот `API_BASE` | должен быть публичный `PANEL_PUBLIC_URL`, панель и cloudflared запущены |

## Для продакшена

cloudflared quick-tunnel — только для теста (адрес временный). Дальше — деплой
панели на сервер с постоянным доменом, HTTPS и PostgreSQL (см. дорожную карту
в корневом README).
